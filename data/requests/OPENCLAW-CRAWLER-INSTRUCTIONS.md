## 概要

PatentMatch.jpの記事作成に必要なデータを収集するクローラー指示書です。
各データリクエスト（REQ-xxx）に基づき、Webクロール・検索・データ整形を行い、
`data/crawled/` 配下にJSON形式で出力してください。

---

## 実行ルール

### Git ワークフロー
1. `main` ブランチから `content/data-crawl-YYYYMMDD` ブランチを作成
2. クロール結果を `data/crawled/` に保存
3. commit & push → PRは不要（直接mainにmergeしてOK）

### 出力フォーマット
- 全て JSON 形式
- UTF-8 エンコーディング
- キーは英語スネークケース（`total_applications` 等）
- 日本語データはそのまま保持
- 各JSONに `metadata` セクションを含める:

```json
{
  "metadata": {
    "request_id": "REQ-20260312-001",
    "crawled_at": "2026-03-14T10:00:00+09:00",
    "sources": ["https://example.com/page1", "https://example.com/page2"],
    "notes": "取得できなかったデータがあればここにメモ"
  },
  "data": {
    // 実際のデータ
  }
}
```

### エラーハンドリング
- サイトがアクセス不可の場合: `"status": "unavailable"` を記録し、検索で代替データを探す
- データが見つからない場合: `"status": "not_found"` を記録し、notesに理由を書く
- 部分的に取得できた場合: 取得できたデータを保存し、欠損部分をnotesに記録

## 各リクエストの詳細指示

### REQ-001: 競合コンテンツ分析

**目的**: PatentMatch.jpの差別化ポイントを見つけるための競合調査

**クロール対象**:
1. `tokkyo.ai` — トップページ + 記事一覧から記事タイトル・カテゴリ・更新頻度を取得
2. `ipbase.go.jp` — 「知財お役立ち情報」セクションのコンテンツ一覧
3. `chizai-times.com` — 記事カテゴリ・人気記事・更新頻度
4. `ipforce.jp` — サービス概要・提供コンテンツの種類

**取得データ**: 各サイトの記事数（概算）、主要カテゴリ、更新頻度、対象読者層、差別化要素

---

### REQ-002: 特許庁統計データ

## クロール時の注意事項

1. **robots.txt を尊重する**: クロール前に各サイトのrobots.txtを確認
2. **アクセス間隔**: 同一サイトへのリクエストは最低3秒間隔を空ける
3. **User-Agent**: `PatentMatch-DataCollector/1.0` を設定
4. **JavaScript レンダリング**: 必要な場合はheadlessブラウザを使用
5. **文字コード**: 日本語サイトはShift_JIS/EUC-JPの場合があるのでUTF-8に変換
6. **PDF/Excel**: 統計資料がPDF/Excelの場合はダウンロードしてテキスト抽出

## 完了報告

各リクエストの処理完了後、対応するYAMLファイルの `status` を `"completed"` に更新してください:

```yaml
status: "completed"
completed_at: "2026-03-14T15:00:00+09:00"
```

取得できなかったデータがある場合は `"partial"` として、notesに詳細を追記してください。

全リクエスト完了後、`git push` して Cowork 側で記事作成を開始します。
